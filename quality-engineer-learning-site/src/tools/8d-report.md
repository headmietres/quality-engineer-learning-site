# 8D 报告（八项纪律问题解决法）

8D（Eight Disciplines，八项纪律）是由福特汽车公司开发的一种**团队导向的系统化问题解决方法论**，用于识别、纠正和消除反复发生的质量问题。8D 的核心思想是：**问题本身并不可怕，可怕的是没有系统的方法去解决它**。每一次质量问题的妥善处理，都是组织学习和能力提升的机会。

## 8D 方法论 8 个步骤详解

### D1：建立团队 (Form the Team)

选择合适的跨职能团队成员是 8D 成功的基石。

| 角色 | 职责 |
|:----|:-----|
| **组长** | 统筹推进、协调资源、管理时间 |
| **技术专家** | 提供技术分析和解决方案的专业意见 |
| **过程工程师** | 分析制造过程、提出工艺改进 |
| **质量工程师** | 统计数据、验证措施效果 |
| **管理层赞助人** | 提供资源支持和决策审批 |

**主要工具**：团队章程、资源需求表、干系人分析

### D2：描述问题 (Describe the Problem)

用 **IS / IS NOT** 框架精确界定问题范围，避免模糊描述。

| 维度 | IS（是什么） | IS NOT（不是什么） |
|:----|:-----------|:-----------------|
| **What（什么）** | 缺陷的物证、类型、代码 | 相似但不相同的缺陷 |
| **Where（地点）** | 发现的位置、产线 | 没有出现的工序/区域 |
| **When（时间）** | 首次发现时间、频次 | 之前正常的时间段 |
| **Who（谁）** | 发现的客户/操作员 | 没有抱怨的客户群 |
| **How Many（数量）** | 不良数量、不良率 | 正常水平的数据 |
| **Trend（趋势）** | 恶化/保持/改善 | 反向的变化趋势 |

**主要工具**：5W2H（What/Where/When/Who/Why/How/How Many）、IS/IS NOT 矩阵、柏拉图

### D3：实施临时遏制措施 (Develop Interim Containment Actions)

在永久措施生效前，**保护客户不受问题影响**。临时措施必须快速、有效、可验证。

典型临时措施：100% 挑选检查、隔离不良批次、切换备用供应商、现场返工返修。

**关键要求**：
- 时间承诺（明确的起止日期）
- 验证每一件流出品（不是抽检，是 100% 检查）
- 明确不良品的处置方式（报废/返工/退回）

**主要工具**：紧急行动计划、隔离通知单、挑选作业指导书

### D4：定义根因 (Define Root Cause)

通过系统化的分析方法，从**可能原因 → 最可能原因 → 根本原因**逐层深入。

| 分析方法 | 用途 |
|:---------|:-----|
| **因果图（鱼骨图）** | 厘清所有潜在原因的类别（5M1E） |
| **5 Why 分析法** | 逐层追问"为什么"，直达根本原因 |
| **FMEA** | 识别失效模式与风险 |
| **DOE 实验设计** | 验证各因子对结果的影响显著性 |
| **数据分层分析** | 按设备/班次/批次/操作员拆分数据 |

**输出**：明确陈述的根因（含物证或数据支撑），以及**根因消失则问题消失**的验证结果。

### D5：选择/验证永久纠正措施 (Choose & Verify Permanent Corrective Actions)

针对 D4 确认的根因，选择能够**从根源消除问题**的永久措施。

- **选择标准**：有效性、可实施性、经济性、副作用
- **验证方法**：小批量试产验证 → A/B 对比 → 加速老化/寿命测试
- **防错优先**：优先采用 Poka-Yoke（防错设计），减少对人的依赖

**主要工具**：决策矩阵（Pugh 矩阵）、DOE、FMEA 更新、控制计划更新

### D6：实施永久措施 (Implement Permanent Corrective Actions)

将经 D5 验证的永久措施正式导入量产。

| 实施内容 | 说明 |
|:---------|:-----|
| **工程变更** | 更新图纸、规格、BOM |
| **工艺变更** | 更新 PFMEA、控制计划、作业指导书 |
| **培训** | 对操作员和检验员进行变更培训 |
| **试生产** | 小批量试产验证变更有效性 |
| **长期监控** | 设置监控指标，追踪措施效果 |

**主要工具**：变更管理流程、培训记录表、控制图（验证措施后过程是否受控）

### D7：预防再发生 (Prevent Recurrence)

将经验教训**横向展开**到类似产品、类似过程、类似系统中，防止同类问题在其他地方重演。

- **经验库更新**：将案例写入 Lessons Learned 数据库
- **FMEA 更新**：将本次问题的失效模式纳入 FMEA
- **控制计划更新**：增加防错/控制手段
- **标准化/文件化**：修订标准作业、检查标准、培训教材
- **体系层面预防**：检查**为何现有体系未能拦截此问题**

### D8：总结/表彰团队 (Conclude & Recognize Team)

完成 8D 全流程后的收尾工作：

1. **问题关闭**：客户确认问题已解决并关闭 8D
2. **成果固化**：将所有文件归档，形成标准化案例
3. **表彰团队**：公开认可团队贡献，提升组织士气
4. **绩效评估**：回顾问题对质量成本、客户满意度的影响

## 8D 报告模板示例

| 项目 | 内容 |
|:-----|:-----|
| **8D 编号** | 8D-2026-001 |
| **问题标题** | XX 型号连接器端子缩针 |
| **客户** | XX 汽车电子有限公司 |
| **产品/批号** | 连接器 8266-01 / 批号 20260115 |
| **严重度** | 重要特性 — 功能丧失 |
| **D1 团队** | 组长：张工；组员：李工(工艺)、王工(模具)、赵工(质量) |
| **D2 问题描述** | 2026/01/15 客户投诉端子缩针，不良率 2.3%（正常 ≤ 0.1%） |
| **D3 临时措施** | 产线 100% 推力测试挑选，已隔离 320 件不良品（完成日期：01/16） |
| **D4 根因** | 模具第 3 腔顶针弹簧疲劳导致退位不足（5 Why + 鱼骨图验证） |
| **D5 永久措施** | 更换弹簧材质（碳钢→不锈钢），寿命从 5 万次提升至 20 万次 |
| **D6 措施实施** | 01/20 完成模具改造，连续试产 500 件零缺陷 |
| **D7 预防** | 更新 PFMEA（增加顶针弹力巡检）、模具保养规程增加弹簧更换周期 |
| **D8 总结** | 问题关闭，客户确认，团队通报表彰 |

> **成功的 8D 不在于报告写得漂亮，而在于根因是否真正被消除。** 如果 8D 做完后问题重复发生，说明 D4（根因分析）或 D5（措施验证）没有做到位。真正有效的 8D，是一个从"救火"到"防火"的组织能力跃迁。
