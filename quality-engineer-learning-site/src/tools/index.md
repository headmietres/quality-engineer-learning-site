# QC 七大手法

QC七大手法（Seven Basic Quality Tools）是质量工程中最基础也最实用的工具集合。由日本质量管理专家石川馨推广，广泛应用于制造业和流程改进。

## 七大手法速览

| 手法 | 用途 | 适用场景 |
|------|------|----------|
| 检查表 (Check Sheet) | 系统化收集数据 | 缺陷记录、每日巡检 |
| 层别法 (Stratification) | 数据分层分析 | 按设备/班次/批次分析 |
| 柏拉图 (Pareto Chart) | 找出关键少数问题 | 80/20 分析、优先排序 |
| 因果图 (Fishbone Diagram) | 分析根因 | 问题根本原因分析 |
| 散布图 (Scatter Diagram) | 分析两组数据关系 | 变量相关性分析 |
| 直方图 (Histogram) | 展示数据分布 | 过程能力初步评估 |
| 控制图 (Control Chart) | 监控过程稳定性 | SPC 核心工具 |

## 检查表 (Check Sheet)

### 定义
检查表（Check Sheet）是一种用于系统化收集和记录数据的结构化表格工具。它帮助质量人员在现场快速、准确地记录质量数据，避免遗漏和主观偏差。

### 用途
- 标准化数据收集流程
- 缺陷类型与频次统计
- 日常巡检与点检记录
- 为后续分析（如柏拉图、直方图）提供原始数据

### 如何制作
1. 明确收集目的和需要回答的问题
2. 确定数据类别和分组方式（如缺陷类型、时间段、班次）
3. 设计简洁易填的表格格式
4. 预留记录人、日期、地点等元信息字段
5. 测试使用并优化表格设计

### 示例
**产品外观缺陷检查表**

| 缺陷类型 | 周一 | 周二 | 周三 | 周四 | 周五 | 合计 |
|---------|:---:|:---:|:---:|:---:|:---:|:---:|
| 划痕 | //// | /// | //// / | // | /// | 17 |
| 色差 | // | / | /// | // | / | 9 |
| 凹陷 | / | // | / | | / | 5 |
| 毛边 | //// | /// | //// | /// | //// | 18 |
| 气泡 | / | | / | / | | 3 |

## 层别法 (Stratification)

### 定义
层别法是将收集到的数据按照不同来源、条件或因素进行分层归类，以便发现数据背后的真实规律和差异。

### 用途
- 识别不同来源的数据差异
- 定位问题发生的具体条件
- 避免数据混杂导致误判
- 为改善行动提供针对性方向

### 如何制作
1. 确定分层维度（如设备编号、操作员、班次、批次、供应商）
2. 按各维度分别汇总数据
3. 计算各层的关键指标（均值、极差、不良率等）
4. 对比层间差异，识别异常层
5. 对异常层深入调查根因

### 示例
某产品经过两台注塑机生产，按设备层别分析不良率：

| 设备编号 | 生产数 | 不良数 | 不良率 |
|---------|:-----:|:-----:|:-----:|
| 注塑机 A | 2000 | 32 | 1.6% |
| 注塑机 B | 2000 | 68 | 3.4% |

分析发现 B 机不良率显著高于 A，应优先排查 B 机的工艺参数或模具状态。

## 柏拉图 (Pareto Chart)

### 定义
柏拉图（Pareto Chart）是基于帕累托原理（80/20 法则）的柱状图与累加折线图的组合图表，用于识别"关键的少数"问题。

### 用途
- 聚焦最重要的质量问题
- 确定改善优先级
- 展示改善前后的对比效果
- 用数据支撑管理决策

### 如何制作
1. 收集各缺陷类别的频次数据
2. 按频次从高到低排序
3. 计算各缺陷的占比和累计占比
4. 绘制柱状图（频次）+ 折线图（累计百分比）
5. 通常关注累计占比前 80% 的少数问题

### 示例
某工序月度缺陷数据：

| 缺陷类型 | 频次 | 占比 | 累计占比 |
|---------|:---:|:---:|:-------:|
| 尺寸超差 | 45 | 37.5% | 37.5% |
| 表面划伤 | 32 | 26.7% | 64.2% |
| 变形 | 18 | 15.0% | 79.2% |
| 材料裂纹 | 12 | 10.0% | 89.2% |
| 其他 | 13 | 10.8% | 100% |

尺寸超差、表面划伤、变形三项合计 79.2%，是优先改善的关键少数。

## 因果图 (Fishbone Diagram / Ishikawa Diagram)

### 定义
因果图又称鱼骨图或石川图，是一种通过系统化梳理问题潜在原因来寻找根本原因的分析工具。

### 用途
- 系统性头脑风暴，找出所有可能原因
- 区分原因类别（5M1E）
- 识别根本原因而非表面现象
- 指导数据收集和验证方向

### 如何制作
1. 明确待分析的问题（写在鱼头）
2. 画出主骨架，标注原因分类（通常用 5M1E）
3. 通过头脑风暴在各分类下添加可能原因
4. 对原因逐层深入（鱼骨分支的次级原因）
5. 筛选最可能的原因，计划验证

### 5M1E 分类
| 分类 | 内容 |
|------|------|
| Man (人员) | 操作技能、培训、疲劳度 |
| Machine (设备) | 设备精度、维护状态、工装 |
| Material (材料) | 来料质量、供应商、批次 |
| Method (方法) | 工艺参数、作业标准 |
| Measurement (测量) | 量具精度、测量方法 |
| Environment (环境) | 温湿度、照明、清洁度 |

## 散布图 (Scatter Diagram)

### 定义
散布图（Scatter Diagram）用于展示两个连续变量之间的关系方向和强度，帮助判断变量之间是否存在相关性。

### 用途
- 分析变量间的相关性（正相关、负相关、无相关）
- 为回归分析提供直观参考
- 识别异常点（离群值）
- 验证因果假设

### 如何制作
1. 确定要分析的两个变量（X 自变量，Y 因变量）
2. 收集至少 30 组配对数据
3. 以 X 为横轴、Y 为纵轴绘制散点
4. 观察点的分布趋势和集中程度
5. 可计算相关系数 r 进行定量判断

### 相关性判断
| 相关系数 r | 相关强度 |
|:---------:|:---------:|
| 0.8 ~ 1.0 | 强正相关 |
| 0.5 ~ 0.8 | 中等正相关 |
| 0.3 ~ 0.5 | 弱正相关 |
| -0.3 ~ 0.3 | 无相关 |
| -0.5 ~ -0.3 | 弱负相关 |
| -0.8 ~ -0.5 | 中等负相关 |
| -1.0 ~ -0.8 | 强负相关 |

## 直方图 (Histogram)

### 定义
直方图以等宽矩形展示连续数据的频数分布，直观呈现数据的集中趋势、离散程度和分布形态。

### 用途
- 观察数据的分布形态（正态、偏态、双峰等）
- 初步评估过程能力
- 识别异常值或数据分层问题
- 与规格限对比，判断是否满足要求

### 如何制作
1. 收集数据（建议 ≥50 个数据点）
2. 计算数据范围 R = Max - Min
3. 确定分组数 k（一般 5~15 组）
4. 计算组距 h = R / k
5. 统计每组频数，绘制柱状图

### 常见分布形态
| 形态 | 可能含义 |
|------|---------|
| 正态分布 | 过程受控，随机变异 |
| 偏态分布 | 存在系统性偏差 |
| 双峰分布 | 数据来自不同总体（需层别分析） |
| 平顶分布 | 多个均值相近的总体混合 |
| 锯齿分布 | 测量分辨率不足或数据修约 |

## 控制图 (Control Chart)

### 定义
控制图（Control Chart）是 SPC 的核心工具，通过绘制过程数据随时间的变化趋势并标注控制限，实时监控过程是否处于统计受控状态。

### 用途
- 区分普通原因和特殊原因变异
- 监控过程稳定性
- 及时发现过程异常，预防不合格品产生
- 评估过程改进效果

### 如何制作
1. 选择适合的控制图类型（详见 SPC 页面）
2. 按时间顺序收集样本数据
3. 计算中心线（CL）和控制限（UCL / LCL）
4. 绘制数据点和控制限
5. 应用判异准则判断过程是否受控

### 详细内容
请参阅 [SPC 统计过程控制](./spc) 页面，了解控制图类型、计算公式和判异八准则。
