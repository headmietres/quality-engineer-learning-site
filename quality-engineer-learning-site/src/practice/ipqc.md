# 过程检验 (IPQC)

IPQC（In-Process Quality Control，过程质量控制）是在产品制造过程中进行的质量控制活动，目的是及时发现和纠正生产过程中的异常，防止批量不良产生。

## 首件检验 (FAI)

### 什么是首件检验

首件检验（First Article Inspection, FAI）是在 **每班次/每个批次/每次换模/每次设备维修后** 生产的第一件（或前几件）产品进行的全面检验。

### 必须做首件的场景

| 场景 | 说明 |
|------|------|
| 开班/换班 | 每班开始生产时 |
| 换模/换线 | 模具或生产线切换后 |
| 换料 | 更换原材料批次或型号 |
| 设备维修后 | 设备关键参数调整或维修后 |
| 工艺变更 | 工艺参数或作业指导书变更后 |
| 停机重启 | 产线停机超过规定时间后重启 |

### 首件检验流程

```
生产准备 → 生产首件 → IPQC检验 → 填写首件报告 → 
判定：合格→批量生产 / 不合格→调整→重新首件
```

### 首件检验内容

| 检查项 | 工具/方法 | 要求 |
|--------|----------|------|
| 外观 | 目视/标准样板 | 符合外观标准 |
| 关键尺寸 | 卡尺/千分尺/投影仪/CMM | 100%测量，记录实测值 |
| 装配性 | 试装/通止规 | 配合良好 |
| 性能参数 | 专用检测设备 | 符合规格要求 |
| 标识 | 目视 | 产品标识、追溯标签正确 |

### 首件检验记录

首件报告应包含：
- 产品名称/图号/工序号
- 设备编号/模具编号
- 检验项目与规格
- 实测数据（精确到小数点后对应精度）
- 检验结论与签名
- **首件样品保留**至该批次生产结束

## 巡检 (Roving Inspection)

### 定义

巡检是指 IPQC 检验员按 **固定频次** 对生产线各工位进行巡回检查，监督工艺纪律执行情况，抽检在制品质量。

### 巡检频次

| 生产类型 | 推荐频次 | 说明 |
|----------|----------|------|
| 大批量流水线 | 每 **1~2 小时** 一次 | 自动化程度高时适当降低 |
| 小批量多品种 | 每批次/每工单一次 | 换型时重点检查 |
| 关键工序 | 每 **30 分钟** 一次 | 如焊接、热处理、注塑 |
| 新员工操作位 | 每 **1 小时** 一次 | 加强指导与监督 |

### 巡检检查表（示例）

| 检查项目 | 检查内容 | 结果 |
|----------|----------|------|
| 5S 管理 | 工作台整洁、物料摆放规范 | ✅/❌ |
| 工艺文件 | 作业指导书（SOP）现行有效，员工按规程操作 | ✅/❌ |
| 设备状态 | 设备参数在工艺规范范围内，无异常报警 | ✅/❌ |
| 工装夹具 | 模具/夹具定位可靠，无磨损 | ✅/❌ |
| 在制品质量 | 抽检产品合格，无批量异常趋势 | ✅/❌ |
| 标识追溯 | 产品标识清晰，数量与流转卡一致 | ✅/❌ |
| 不合格品 | 不合格品已隔离且有标识，无混入 | ✅/❌ |

## 过程能力监控

### Cp / Cpk 现场应用

过程能力指数反映了生产过程满足规格要求的能力。

| 指数 | 含义 | 现场行动 |
|------|------|----------|
| Cpk ≥ 1.67 | 过程能力充裕 | 维持现状，可考虑优化降低成本 |
| 1.33 ≤ Cpk < 1.67 | 过程能力尚可 | 正常控制，持续监控 |
| 1.00 ≤ Cpk < 1.33 | 过程能力不足 | 加强监控，启动改进计划 |
| Cpk < 1.00 | 过程能力严重不足 | 立即停产整改 |

### 现场过程能力监控要点

1. **数据真实**：测量数据必须来自实际生产，不得人为筛选
2. **样本代表性**：覆盖不同班次、设备、操作员
3. **动态更新**：定期（每周/每月）更新过程能力分析
4. **异常追溯**：Cpk 突然下降时追溯至具体工序和原因

## 控制图现场应用

### 常用控制图类型

| 控制图 | 数据类型 | 应用场景 |
|--------|----------|----------|
| **X̄-R 图** | 计量型 | 每班/每批次测量数据，n≤10 |
| **X̄-S 图** | 计量型 | 样本量较大（n>10），或使用计算机采集 |
| **I-MR 图** | 计量型 | 单件生产、自动检测、破坏性检验 |
| **p 图** | 计件型 | 不合格品率控制 |
| **np 图** | 计件型 | 不合格品数控制（样本量恒定） |
| **c 图** | 计点型 | 单位产品缺陷数控制 |
| **u 图** | 计点型 | 单位面积/长度缺陷数控制 |

### 控制图八大判异规则（Western Electric Rules）

| 编号 | 规则 | 含义 |
|------|------|------|
| 1 | 1 点超出控制界限（UCL/LCL） | 过程失控 |
| 2 | 连续 3 点中有 2 点落在 A 区或以上 | 过程均值漂移 |
| 3 | 连续 5 点中有 4 点落在 B 区或以上 | 过程有偏移趋势 |
| 4 | 连续 7 点落在中心线同侧 | 过程均值已偏移 |
| 5 | 连续 7 点呈上升或下降趋势 | 过程存在趋势性变化 |
| 6 | 连续 14 点交替上下 | 存在两个不同过程混合 |
| 7 | 连续 8 点落在 C 区（中心线附近） | 过程分层，数据可能造假 |
| 8 | 连续 15 点落在 C 区 | 过程变差异常变小，需确认测量系统 |

### 现场控制图常见问题

| 问题 | 原因 | 对策 |
|------|------|------|
| 点超出控制限 | 过程异常（刀具磨损、模具松动等） | 立即停机排查 |
| 连续上升/下降 | 工具磨损、温度变化、材料批次差异 | 分析趋势原因 |
| 数据分层 | 不同班次/设备数据混在同一控制图 | 分层绘制控制图 |
| 控制限过宽 | 组内变差大或组间变差小 | 调整分组方式 |

## 异常处理流程

```
IPQC发现异常 → 暂停生产 → 标识隔离异常品 → 
填写《品质异常报告》 → 通知相关部门 → 
原因分析 → 制定纠正措施 → 实施验证 → 
合格→恢复生产 / 不合格→继续整改
```

### 异常分级处理

| 级别 | 定义 | 响应时间 | 处置方式 |
|------|------|----------|----------|
| **一级异常** | 轻微个别缺陷，不影响功能 | 4 小时内 | 现场指导纠正 |
| **二级异常** | 批量问题或影响功能 | 2 小时内 | 停线、评审、返工 |
| **三级异常** | 严重质量问题或安全隐患 | 立即 | 停线、追溯隔离、启动 8D |

### IPQC 停线权限

IPQC 检验员有权在以下情况要求生产线停线：
- 发现批量不合格（连续 5 件以上）
- 发现可能影响安全或关键功能的问题
- 工艺参数严重偏离标准
- 首件检验不合格但生产已开始

---

> **核心原则**：IPQC 不是挑出不良品，而是通过过程控制让不良品 **不发生**。预防胜于检验。
