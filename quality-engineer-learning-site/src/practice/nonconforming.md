# 不合格品处理

不合格品（Nonconforming Product）是指不满足规定要求的产品。不合格品处理是质量管理体系中至关重要的环节，直接关系到产品质量、成本和客户满意度。

## 不合格品标识与隔离

### 标识要求

所有不合格品必须 **清晰、醒目、唯一** 地标识，防止与合格品混淆。

| 标识方式 | 适用场景 | 要求 |
|----------|----------|------|
| 红色标签 | 退货/报废品 | 标签上注明不合格原因、日期、检验员 |
| 黄色标签 | 待评审品 | 等待 MRB 评审处置 |
| 红色区域/料箱 | 现场隔离区 | 集中存放不合格品 |
| 系统标识 | ERP/MES 系统 | 批次状态锁定，不可流转 |

### 隔离要求

```
发现不合格 → 立即停止流转 → 移入不合格品隔离区 → 
挂牌标识 → 记录台账 → 等待评审
```

### 隔离方式

| 隔离方式 | 说明 |
|----------|------|
| **物理隔离** | 划定专门的不合格品区域，用围栏/标线分隔 |
| **料箱隔离** | 使用红色/黄色料箱盛放，与合格品料箱区分 |
| **系统隔离** | 在 ERP/MES 中锁定该批次，禁止流转和出货 |

### 隔离区管理要点

- 隔离区应位于可见位置，便于监管
- 隔离区不得存放合格品
- 隔离区物品应在 **规定期限内** 完成评审处置
- 每日盘点隔离区台账，防止遗漏

## MRB 评审流程

MRB（Material Review Board，物料评审委员会）是由质量、工程、生产、采购等跨职能团队组成的评审组织，决定不合格品的处置方式。

### MRB 组织架构

| 角色 | 部门 | 职责 |
|------|------|------|
| **主席** | 质量经理 | 召集会议，主持评审，签署最终决定 |
| **成员** | 工程/技术 | 评估功能影响及返工可行性 |
| **成员** | 生产/制造 | 评估返工成本与周期 |
| **成员** | 采购/销售 | 协调供应商或客户意见 |
| **记录员** | 质量工程师 | 记录评审结论，跟踪关闭 |

### MRB 评审流程

```
不合格品报告 → 召集MRB会议 → 
各部门评审 → 形成处置意见 → 
质量经理批准 → 执行处置 → 
验证关闭
```

### MRB 决策树

```
不合格品发生
    │
    ├── 是否可通过返工达到合格？──→ 是 → 返工 (Rework)
    │
    ├── 是否不影响功能且客户同意？──→ 是 → 让步接收 (Concession)
    │
    ├── 是否可通过降级使用？──→ 是 → 降级 (Downgrade)
    │
    └── 以上均不可行？──→ 报废 (Scrap)
```

## 四种处置方式

### 1. 让步接收 (Concession)

**定义**：对使用或放行一定数量的不符合规定要求的产品（包括返工后仍不完全合格的产品）的许可。

**适用条件**：
- 不影响产品安全性、关键功能
- 不影响装配和后续工序
- **获得客户书面同意**（客户特殊要求时必须）
- 有明确的限制条件（数量、时间范围）

**流程**：
```
使用部门申请 → 工程评估 → 质量确认 → 
客户批准（如需） → 让步接收标签 → 可追溯记录
```

### 2. 返工 (Rework)

**定义**：为使不合格品符合要求而对其采取的措施。

**常见返工类型**：

| 类型 | 说明 | 验证方式 |
|------|------|----------|
| 全检挑选 | 从批中挑出合格品 | 100% 检验 |
| 返修加工 | 补充加工（如去毛刺、重焊） | 重新检验 |
| 拆解重装 | 拆解后重新组装 | 全功能测试 |
| 清洗/返烤 | 表面处理 | 外观+性能测试 |

**返工管理要点**：
- 返工需有 **返工作业指导书**
- 返工后必须 **重新检验**
- 返工批次的 **可追溯性** 要完整记录
- 返工后的产品按 **正常流程重新入库**

### 3. 报废 (Scrap)

**定义**：对不合格品进行销毁处理，使其无法再使用。

**适用条件**：
- 无法通过返工达到合格标准
- 让步接收不被客户认可
- 降级使用无市场价值
- 安全隐患或法律法规强制报废

**报废流程**：
```
MRB决议报废 → 报废申请审批 → 
破坏性处理（销毁） → 财务核销 → 记录存档
```

### 4. 降级 (Downgrade)

**定义**：将不合格品用于较低等级的应用场景。

**适用条件**：
- 有明确的降级标准（如外观等级从 A 级降为 B 级）
- 不影响降级后的使用功能
- 有对应的销售渠道或内部使用场景

**管理要求**：
- 降级产品需 **明确标识降级等级**
- 降级产品不得冒充正品出货
- 需在 COA 或出货报告中注明降级状态

## NCR 开具与跟踪

NCR（Nonconformance Report，不合格报告）是对不合格事件进行记录、追踪和关闭的正式文件。

### NCR 内容

| 字段 | 说明 |
|------|------|
| NCR 编号 | 唯一标识号（如 NCR-2026-0001） |
| 发现日期/地点 | 发现时间、工序/工位 |
| 产品信息 | 产品名称、图号、批次号、数量 |
| 不合格描述 | 具体不良现象，附照片或检测数据 |
| 不合格原因 | 初步原因分析 |
| 处置方式 | MRB 评审结论 |
| 纠正措施 | 针对根因的整改措施 |
| 验证结果 | 措施有效性的验证 |
| 关闭日期 | 所有活动完成后的关闭日期 |

### NCR 生命周期

```
开立 (Open) → 评审 (Review) → 处置 (Disposition) → 
整改 (Action) → 验证 (Verification) → 关闭 (Closed)
```

### NCR 跟踪要求

- **台账管理**：建立 NCR 台账，定期更新状态
- **超期预警**：超过规定时限未关闭的 NCR 自动升级至管理层
- **统计分析**：每月统计 NCR 数量、类型分布、On-time 关闭率
- **趋势分析**：识别高频不合格类型，推动系统性改善

## 纠正措施 (CA) 与预防措施 (PA)

### 纠正措施 (Corrective Action, CA)

**定义**：为消除已发现的不合格或其他不期望情况的原因所采取的措施。

**步骤（基于 8D 方法论）**：

| 步骤 | 内容 | 工具 |
|------|------|------|
| D1 | 组建团队 | 跨职能小组 |
| D2 | 描述问题 | 5W2H |
| D3 | 临时措施 | 围堵、筛选 |
| D4 | 根本原因分析 | 鱼骨图、5 Why |
| D5 | 永久纠正措施 | FMEA |
| D6 | 实施与验证 | 试运行、数据对比 |
| D7 | 预防再发 | 更新 FMEA/CP/SOP |
| D8 | 总结表彰 | 横向展开 |

### 预防措施 (Preventive Action, PA)

**定义**：为消除潜在不合格或其他潜在不期望情况的原因所采取的措施。

**与纠正措施的区别**：

| 维度 | 纠正措施 (CA) | 预防措施 (PA) |
|------|--------------|--------------|
| 触发时机 | 问题 **已发生** | 问题 **尚未发生** |
| 驱动来源 | 客诉、NCR、内部不合格 | FMEA、趋势分析、风险评审 |
| 例子 | 因为尺寸超差而修改刀具更换频率 | 根据刀具寿命数据提前规划更换周期 |

### CAPA 系统管理

CAPA（纠正与预防措施）体系是质量管理体系的核心组成部分：

```
问题识别 → 风险评估 → 原因分析 → 
措施制定 → 实施执行 → 效果验证 → 
标准化 → 横向展开 → 关闭
```

### CAPA 有效性评估

| 评估维度 | 指标 | 目标 |
|----------|------|------|
| 及时性 | CAPA 按时关闭率 | ≥ 90% |
| 有效性 | 同类问题重复发生率 | ≤ 5% |
| 闭环率 | 验证通过率 | ≥ 95% |
| 系统性 | SOP/FMEA 更新率 | 100% |

---

> **核心原则**：不合格品不是终点，而是改进的起点。每一次不合格处理都是让质量体系更强大的机会。
