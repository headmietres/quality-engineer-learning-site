# PPAP 生产件批准程序

PPAP（Production Part Approval Process，生产件批准程序）是用于确认供应商的生产过程是否已准备好批量生产、是否能持续满足客户所有要求的标准化流程。PPAP 起源于美国汽车工业，现已被 IATF 16949 体系广泛采用，成为全球制造业供应链质量管理的核心工具。

## PPAP 的目的

PPAP 的核心目标是在**正式量产前**验证以下事项：

1. **设计完整性** — 产品和设计记录满足客户要求
2. **过程能力** — 生产工艺具备稳定满足规格的能力
3. **测量系统可靠性** — 检验/测试方法经过 MSA 验证
4. **供应链可追溯** — 所有分供方件也得到同等管控
5. **批量一致性** — 试产样件能代表批量生产的质量水平

## 18 项提交要素

PPAP 第 4 版要求提交以下 18 项要素：

| 编号 | 要素 | 说明 |
|:----:|:-----|:-----|
| 1 | 设计记录 | 产品图纸、数模、技术规范 |
| 2 | 工程变更文件 | 授权变更的工程批准文件 |
| 3 | DFMEA | 设计失效模式分析 |
| 4 | 过程流程图 | 生产工艺步骤与检验点 |
| 5 | PFMEA | 过程失效模式分析 |
| 6 | 控制计划 | 各工序控制方法、频次、反应计划 |
| 7 | MSA 结果 | 量具重复性与再现性研究 |
| 8 | 尺寸结果 | 全尺寸检测报告 |
| 9 | 材料/性能测试结果 | 材质报告、性能试验报告 |
| 10 | 初始过程能力研究 | Cp/Cpk 或 Ppk 分析 |
| 11 | 实验室资质 | 测试实验室的认证范围 |
| 12 | 外观批准报告 (AAR) | 颜色、纹理、光泽等外观认可 |
| 13 | 散装材料要求检查表 | 仅适用于散装材料（如化学品） |
| 14 | 样品产品 | 提交代表性样件 |
| 15 | 标准样品 | 作为比较基准的封样 |
| 16 | 检查辅具 | 检具/量具的验证记录 |
| 17 | 客户特殊要求 | 客户指定的额外要求 |
| 18 | 零件提交保证书 (PSW) | PPAP 最终签署文件 |

## 5 个提交等级

客户根据产品风险和供应商成熟度指定提交等级：

| 等级 | 名称 | 说明 | 典型场景 |
|:----:|:-----|:-----|:---------|
| **Level 1** | 仅 PSW | 仅提交零件提交保证书 | 低风险变更、成熟供应商 |
| **Level 2** | PSW + 样品 | 提交 PSW + 有限支持数据 + 样品 | 简单零件、小批量 |
| **Level 3** | 全部要素 | **提交全部 18 项要素**（最常用） | 新零件、新供应商（默认等级） |
| **Level 4** | 客户指定 | 由客户定义需要提交的要素 | 特殊合同要求 |
| **Level 5** | PSW + 现场评审 | 提交 PSW + 供应商现场评审资料 | 高风险零件、新供应商的首次提交 |

> **注**：如果客户未指定提交等级，供应商默认按 **Level 3** 准备全部 18 项要素。

## 关键/重要/一般特性定义

PPAP 要求对产品特性进行分类管理：

| 特性类别 | 定义 | 管控要求 |
|:---------|:-----|:---------|
| **关键特性** (Critical, ⚠) | 涉及安全或法规符合性，失效会导致人身伤害或法规违反 | 100% 检验或防错控制，Ppk ≥ 1.67 |
| **重要特性** (Significant, ◆) | 影响功能、配合、性能，失效会导致功能丧失或重大客户抱怨 | SPC 监控，Ppk ≥ 1.33 |
| **一般特性** (General) | 不影响功能和安全的一般外观或尺寸要求 | 常规检验，Ppk ≥ 1.00 |

## 初始过程能力研究要求

PPAP 要求在新项目量产前进行初始过程能力研究（Initial Process Capability Study）：

- **取样方式**：连续生产 100~300 件（通常 125 件），涵盖正常生产中可能出现的变异
- **稳定性前提**：分析过程能力前，必须先确认过程处于统计受控状态（控制图无异常）
- **评估标准**：
  - **Ppk ≥ 1.67** — 过程能力满足要求，可批准量产
  - **1.33 ≤ Ppk < 1.67** — 过程能力可接受，但需加强监控和持续改进
  - **Ppk < 1.33** — 过程能力不足，必须制定和实施改进计划，不得直接批准量产
- **长期追踪**：PPAP 批准后，供应商需在量产中持续监控过程能力，定期向客户提交 Cp/Cpk 报告

> **PPAP 是一个流程，不是一个事件。** 它贯穿于新项目开发的整个过程，从设计评审到工装验证再到批量试产。成功的 PPAP 不是靠一次性的资料准备，而是靠从一开始就嵌入到项目各阶段的质量策划和过程控制。
